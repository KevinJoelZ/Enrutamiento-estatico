Tema 05

Verificación y Diagnóstico

Comandos y métodos para comprobar que las rutas estáticas funcionan correctamente y resolver fallos de conectividad entre redes.

Temas

Comandos de verificación en el router

Tabla de enrutamiento completa

Router# show ip route

Muestra todas las rutas conocidas. Las rutas estáticas aparecen marcadas con una S al inicio de la línea. Ejemplo esperado en Router0: 

S    192.168.200.0/24 [1/0] via 192.168.110.2
S    192.168.210.0/24 [1/0] via 192.168.205.2
C    192.168.100.0/24 is directly connected, GigabitEthernet0/0
C    192.168.110.0/24 is directly connected, GigabitEthernet0/1
C    192.168.205.0/24 is directly connected, GigabitEthernet0/2

Solo rutas estáticas

Router# show ip route static

Configuración activa

Router# show running-config | section ip route
Router# show ip interface brief

El primer comando filtra únicamente las líneas de rutas estáticas. El segundo confirma que las interfaces están up/up y tienen la IP correcta.

Pruebas de conectividad

Desde el router

Router# ping 192.168.200.5
Router# ping 192.168.210.8
Router# traceroute 192.168.200.5

Desde los PCs (Packet Tracer)

Abre el PC → pestaña Desktop → Command Prompt:

C:\> ping 192.168.200.5    ! PC0 (Red 1) hacia PC1 (Red 2)
C:\> ping 192.168.210.8    ! PC0 (Red 1) hacia PC2 (Red 3)
C:\> tracert 192.168.210.8

Interpretación de códigos en la tabla

Código Significado
C Red conectada directamente (Connected)
S Ruta estática (Static)
R RIP (dinámico)
O OSPF (dinámico)
[1/0] [distancia administrativa/métrica]

Errores frecuentes y solución

Síntoma Causa probable Solución
Ping falla entre LANs Falta ruta estática en algún router Revisar show ip route en cada router del camino
Ruta estática no aparece Next-hop inalcanzable Verificar IP del enlace WAN y que la interfaz esté up
Ping OK en router, falla en PC Gateway incorrecto en el host Configurar default gateway del PC al router local
Interfaz administratively down No se aplicó no shutdown Entrar a la interfaz y ejecutar no shutdown
Next-hop incorrecto Confusión entre routers vecinos Consultar tabla de enlaces WAN en la topología

Checklist de verificación

  • Todas las interfaces tienen IP y están up/up.
  • Cada router tiene rutas hacia las dos redes remotas.
  • Los PCs tienen IP, máscara y gateway correctos.
  • ping funciona entre hosts de distintas LANs.
  • tracert muestra el camino esperado por los routers.
