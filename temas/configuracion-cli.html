Tema 03

Configuración por Línea de Comandos

Comandos Cisco IOS para configurar interfaces y rutas estáticas en Router0, Router1 y Router2 del escenario de laboratorio.

Temas

01 · Fundamentos 02 · Topología IP 03 · Configuración CLI 04 · Interfaz gráfica 05 · Verificación 06 · Laboratorio

Sintaxis del comando

Router(config)# ip route <red_destino> <máscara> <next-hop> [distancia_administrativa]

El comando se ejecuta en modo de configuración global. La distancia administrativa es opcional; si no se indica, el valor por defecto es 1.

Router0 — Red 1

Gateway de la LAN 192.168.100.0/24. Conectado a Router2 y Router1 por enlaces WAN.

enable
configure terminal
hostname Router0

! Interfaz LAN Red 1
interface GigabitEthernet0/0
 ip address 192.168.100.1 255.255.255.0
 no shutdown
 exit

! Enlace hacia Router2 (192.168.110.0/24)
interface GigabitEthernet0/1
 ip address 192.168.110.1 255.255.255.0
 no shutdown
 exit

! Enlace hacia Router1 (192.168.205.0/24)
interface GigabitEthernet0/2
 ip address 192.168.205.1 255.255.255.0
 no shutdown
 exit

! Rutas estáticas hacia redes remotas
ip route 192.168.200.0 255.255.255.0 192.168.110.2
ip route 192.168.210.0 255.255.255.0 192.168.205.2

end
write memory

Router2 — Red 2

Gateway de la LAN 192.168.200.0/24. Router central del triángulo.

enable
configure terminal
hostname Router2

interface GigabitEthernet0/0
 ip address 192.168.200.4 255.255.255.0
 no shutdown
 exit

interface GigabitEthernet0/1
 ip address 192.168.110.2 255.255.255.0
 no shutdown
 exit

interface GigabitEthernet0/2
 ip address 192.168.150.1 255.255.255.0
 no shutdown
 exit

ip route 192.168.100.0 255.255.255.0 192.168.110.1
ip route 192.168.210.0 255.255.255.0 192.168.150.2

end
write memory

Router1 — Red 3

Gateway de la LAN 192.168.210.0/24.

enable
configure terminal
hostname Router1

interface GigabitEthernet0/0
 ip address 192.168.210.7 255.255.255.0
 no shutdown
 exit

interface GigabitEthernet0/1
 ip address 192.168.150.2 255.255.255.0
 no shutdown
 exit

interface GigabitEthernet0/2
 ip address 192.168.205.2 255.255.255.0
 no shutdown
 exit

ip route 192.168.100.0 255.255.255.0 192.168.205.1
ip route 192.168.200.0 255.255.255.0 192.168.150.1

end
write memory

Resumen de rutas estáticas

Router Red destino Next-hop
Router0 192.168.200.0/24 192.168.110.2
Router0 192.168.210.0/24 192.168.205.2
Router2 192.168.100.0/24 192.168.110.1
Router2 192.168.210.0/24 192.168.150.2
Router1 192.168.100.0/24 192.168.205.1
Router1 192.168.200.0/24 192.168.150.1

Configura primero las interfaces y luego las rutas. Un next-hop debe ser alcanzable; de lo contrario, la ruta aparecerá como inactiva en la tabla.

Configuración de hosts (PCs)

Cada PC debe tener IP, máscara y gateway por defecto apuntando al router de su LAN:

! PC0 — Red 1
IP Address:  192.168.100.2
Subnet Mask: 255.255.255.0
Default GW:  192.168.100.1

! PC1 — Red 2
IP Address:  192.168.200.5
Subnet Mask: 255.255.255.0
Default GW:  192.168.200.4

! PC2 — Red 3
IP Address:  192.168.210.8
Subnet Mask: 255.255.255.0
Default GW:  192.168.210.7
