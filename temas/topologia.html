Tema 02

Topología y Direccionamiento IP

Escenario de laboratorio con tres redes LAN conectadas en malla triangular mediante tres routers Cisco 2911 en Packet Tracer.

Temas

01 · Fundamentos 02 · Topología IP 03 · Configuración CLI 04 · Interfaz gráfica 05 · Verificación 06 · Laboratorio

Visión general del escenario

El laboratorio implementa tres segmentos de red (Red 1, Red 2 y Red 3), cada uno con un switch 2960, hosts finales (PC y laptops) y un router 2911 como gateway. Los routers se interconectan formando una topología triangular, lo que obliga a configurar rutas estáticas en cada equipo para alcanzar redes remotas.

Redes LAN (segmentos de usuario)

Red Dirección Gateway Router Hosts de ejemplo
Red 1 192.168.100.0/24 192.168.100.1 Router0 · Gig0/0 PC0 (.2), Laptop0 (.3), PC3 (.4)
Red 2 192.168.200.0/24 192.168.200.4 Router2 · Gig0/0 PC1 (.5), Laptop1 (.6), PC4 (.7)
Red 3 192.168.210.0/24 192.168.210.7 Router1 · Gig0/0 PC2 (.8), Laptop2 (.9), PC5 (.10)

Enlaces entre routers (WAN)

Enlace Red Router A Router B
Router0 ↔ Router2 192.168.110.0/24 Router0 Gig0/1 → .1 Router2 Gig0/1 → .2
Router2 ↔ Router1 192.168.150.0/24 Router2 Gig0/2 → .1 Router1 Gig0/1 → .2
Router0 ↔ Router1 192.168.205.0/24 Router0 Gig0/2 → .1 Router1 Gig0/2 → .2

Flujo de tráfico entre redes

Si PC0 (192.168.100.2) envía un paquete a PC1 (192.168.200.5), el flujo es el siguiente:

  1. PC0 envía el paquete a su gateway 192.168.100.1 (Router0).
  2. Router0 consulta su tabla y encuentra la ruta hacia 192.168.200.0/24 vía 192.168.110.2.
  3. Router2 recibe el paquete y lo entrega directamente a la LAN Red 2 por Gig0/0.
  4. El switch reenvía el tráfico hacia PC1.

Cada router debe conocer las redes que no tiene conectadas directamente. Sin rutas estáticas, solo habría comunicación dentro de cada LAN local.
