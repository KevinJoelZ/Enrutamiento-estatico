Tema 04

Interfaz Gráfica en Packet Tracer

Alternativa visual al CLI: configuración de interfaces y rutas estáticas desde la pestaña Config de cada router, con equivalencia en comandos IOS.

Temas

01 · Fundamentos 02 · Topología IP 03 · Configuración CLI 04 · Interfaz gráfica 05 · Verificación 06 · Laboratorio

Acceder a la configuración gráfica

En Cisco Packet Tracer, haz clic sobre un router y abre la ventana de propiedades. Selecciona la pestaña Config. En el panel izquierdo encontrarás las secciones:

  • GLOBAL — ajustes generales del dispositivo.
  • ROUTING / ENRUTAMIENTO → Estático — tabla de rutas estáticas.
  • INTERFACE — configuración IP de cada puerto GigabitEthernet.

Configurar una interfaz (GUI)

Para asignar IP a un enlace WAN, selecciona la interfaz deseada (por ejemplo GigabitEthernet0/1), activa el puerto y completa los campos IPv4:

Campo GUI Valor ejemplo (Router0 Gig0/1)
Port Status On
IPv4 Address 192.168.110.1
Subnet Mask 255.255.255.0

Agregar rutas estáticas (GUI)

En el menú ENRUTAMIENTO → Estático, completa los campos y pulsa Agregar:

  • Red: dirección de red destino (ej. 192.168.200.0).
  • Máscara: máscara de subred (ej. 255.255.255.0).
  • Siguiente salto: IP del router vecino (ej. 192.168.110.2).

Comandos IOS equivalentes

Packet Tracer muestra en la parte inferior de la ventana Config los Equivalent IOS Commands. Esto permite relacionar cada acción gráfica con su comando CLI correspondiente, facilitando el aprendizaje de ambos métodos. 

! Lo que agregas en GUI se traduce a:
Router(config)# ip route 192.168.200.0 255.255.255.0 192.168.110.2
Router(config)# ip route 192.168.210.0 255.255.255.0 192.168.205.2

! Configuración de interfaz en GUI equivale a:
Router(config)# interface GigabitEthernet0/1
Router(config-if)# ip address 192.168.110.1 255.255.255.0
Router(config-if)# no shutdown

La GUI y el CLI producen el mismo resultado. Se recomienda dominar ambos: la interfaz gráfica acelera el laboratorio; el CLI es el estándar en equipos reales.
